# Contributing

Thank you for your interest in improving Graph Admin Studio.

## How to Contribute

- **Bug reports:** Open an issue with clear reproduction steps and expected vs. actual behavior.
- **Documentation:** Improvements to clarity, accuracy, and onboarding are always welcome.
- **Feature proposals:** Open an issue before a PR to discuss scope and fit.
- **Code contributions:** Submit a pull request with a focused change.

## Design Philosophy Alignment

Graph Admin Studio prioritizes:

- Graph-first execution with PowerShell fallback
- Deterministic behavior (no hidden or probabilistic actions)
- Auditability and evidence-first outputs
- Safety classification (safe / caution / dangerous)
- Local-first operation and transparency

Changes that conflict with these principles are unlikely to be accepted.

## PR Expectations

- Clear description of intent and impact
- Tests where applicable
- No breaking changes without prior discussion
- Maintain readability and explainability

## Review Process

All PRs are reviewed by the maintainer. Feedback may request iteration. Stale PRs may be closed.
